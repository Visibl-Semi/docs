# Variables

Variables allow you to create dynamic content that can be reused throughout your documentation.

## Basic Usage

Variables are defined and used with specific syntax:

```markdown
{{variable_name}}
```

## Common Variables

### User Information
- `{{user.name}}` - User's display name
- `{{user.email}}` - User's email address
- `{{user.role}}` - User's role/permission level

### Environment Variables
- `{{env.api_url}}` - API endpoint URL
- `{{env.version}}` - Current version
- `{{env.region}}` - User's region

## Examples

### API Documentation

```markdown
Make a request to {{env.api_url}}/users/{{user.id}}
```

### Configuration

```yaml
api_key: {{user.api_key}}
endpoint: {{env.api_url}}
```

## Best Practices

<Tip>
Use descriptive variable names and provide fallback values where appropriate.
</Tip>

- Keep variable names consistent
- Document available variables
- Test with different user contexts
- Provide meaningful defaults
