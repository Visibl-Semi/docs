## Install

```bash
curl -fsSL https://visiblsemi.com/api/install | bash
```

The installer:
- Downloads the latest CLI
- Installs to `/usr/local/bin` or `~/.visibl/bin`
- Configures shell PATH
- Auto-detects local LLM services

## Verification

```bash
visibl
```

This launches the TUI and tests the connection to your LLM service.

## Configuration

### Automatic

The CLI auto-discovers LLM services on:
- `localhost:58180` (default)
- Systemd service: `visibl-llm`
- Docker container: `visibl-llm`

### Manual

```bash
# Set endpoint
visibl config set-llm-endpoint 192.168.1.100:58180

# Or use environment variables
export VISIBL_LLM_HOST=192.168.1.100
export VISIBL_LLM_PORT=58180
```

Configuration stored in `~/.config/visibl/visibl.json`.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Connection Issues">
    Check LLM service:
    ```bash
    visibl-smi
    ```
    
    Test CLI connection:
    ```bash
    visibl
    ```
    
    The CLI auto-discovers the service. Connection errors will be shown in the output.
  </Accordion>
  
  <Accordion title="PATH Issues">
    Add to PATH:
    ```bash
    export PATH="$HOME/.visibl/bin:$PATH"
    echo 'export PATH="$HOME/.visibl/bin:$PATH"' >> ~/.bashrc
    source ~/.bashrc
    ```
  </Accordion>

    - Output from running `visibl`
    - Screenshot from `visibl-smi`
    - Description of issue
  </Accordion>
</AccordionGroup>
