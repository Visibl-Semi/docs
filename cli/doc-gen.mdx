Generate comprehensive AI-powered documentation for your RTL modules. The Doc Agent analyzes your RTL code and produces professional documentation with interface tables, timing diagrams, and detailed descriptions.

## Quick Start

### Interactive TUI Mode

```bash
visibl

# Generate for specific module
/docs uart_core
```

## Output

Documentation is generated in two formats:

```
your-project/docs/
├── markdown/
│   └── uart_core.visibl.md      # Version control friendly
├── googledocs/
│   └── uart_core.docx            # Design review ready
└── logs/
    └── docagent_docs-*.log       # Debug logs
```

**Markdown**: Interface tables, signal descriptions, functional behavior, protocol specifications

**DOCX**: Rich formatting with timing diagrams, waveforms, block diagrams, and complete interface tables

## CI/CD Integration

Automate documentation generation in your pipeline:

```yaml
# .github/workflows/docs.yml
name: Update Documentation
on:
  push:
    paths: ['rtl/**/*.sv', 'rtl/**/*.v']
  
jobs:
  docs:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Generate Documentation
        run: |
          visibl docs uart_core
          git add docs/
          git commit -m "Auto-update docs" || exit 0
          git push
```

## Troubleshooting

If documentation generation fails, the logs contain detailed error information:

```bash
# Logs are stored in your project
docs/logs/docagent_docs-*.log

# View the most recent log
ls -lt docs/logs/ | head -n 2
cat docs/logs/docagent_docs-<timestamp>.log
```

Send the log file to [hi@visiblsemi.com](mailto:hi@visiblsemi.com) along with:
- The module name you tried to document
- Your RTL file path
- Any error messages shown in the CLI
